# TDD

This README describes only the `tdd/` skill directory under `AI/AI-Coding`.

## Purpose

`tdd` enforces a red-green-refactor workflow for feature delivery and bug fixing. It emphasizes tracer bullets, minimal passing implementations, and testable interface design.

## Directory Structure

```text
tdd/
  README.md
  SKILL.md
  deep-modules.md
  interface-design.md
  mocking.md
  refactoring.md
  tests.md
```

## Files

### `SKILL.md`

The main skill definition. It drives the agent through a disciplined TDD loop with explicit checkpoints.

### `tests.md`

Examples and rules for good test selection, naming, and scope.

### `mocking.md`

Guidance on when and how to mock without collapsing test value.

### `deep-modules.md`, `interface-design.md`, `refactoring.md`

Supporting references for designing testable modules and refactoring safely after green tests.
