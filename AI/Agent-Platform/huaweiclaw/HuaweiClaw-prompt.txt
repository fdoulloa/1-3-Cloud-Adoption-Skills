Estás ayudándome a construir HuaweiClaw, un agente de IA personal creado desde cero que funciona localmente y usa Telegram, Whatsapp y Web como interfases.

No es un fork ni depende de OpenClaw. Es una implementación propia, simple, segura y completamente bajo mi control.

Objetivo

Crear un agente que pueda:
	•	Comunicarse conmigo vía Telegram como principal y Web tipo ChatGPT
	•	Pensar usando LLMs
	•	Ejecutar herramientas
	•	Recordar información de forma persistente
	•	Ejecutarse completamente en local

Requisitos obligatorios
	•	Basado en JavaScript con TypeScript (ES modules)
	•	Arquitectura modular y clara
	•	Servidor web para interfaz web
	•	Usar webhooks para Telegram
	•	Whitelist de Telegram user ID
	•	Credenciales y parametrizacion de llm y mcp en .env
	•	Memoria persistente con mongoDB
	•	Agent loop con límite de iteraciones
	•	Sin skills externas no verificadas
	•	Seguridad como prioridad
	•	Compuesto por 7 subagentes: 
		•	Agente orquestador, que clasifica la tarea, selecciona el agente y sintetiza respuesta
		•	Agente de negocio que tiene acceso a herramientas de negocio
		•	Agente de datos que tiene acceso a herramientas de datos
		•	Agente de codigo que tiene acceso a herramientas de codigo
		•	Agente de comunicacion que tiene acceso a herramientas de comunicacion
		•	Agente de conocimiento que tiene acceso a herramientas RAG y busqueda de informacion en internet
		•	Agente huaweicloud que tiene acceso a herramientas de huaweicloud
	• Comunicacion entre agentes por A2A (agent to agent)
	• Cada agente puede tener un LLM diferente
	• Cada agente debe tener sus propias Credenciales y herramientas
	• El patrón tiene tres capas: el orquestador que enruta, los agentes especializados que ejecutan, y un sandbox seguro que controla el acceso a herramientas 	

Principios de seguridad clave
	•	Mínimo privilegio — cada agente solo tiene acceso a las herramientas que necesita para su dominio. El agente de código no puede enviar emails, el de comunicaciones no puede ejecutar SQL.
	•	Sandbox de ejecución — si usas run_python, ejecútalo dentro de un contenedor Docker o con RestrictedPython. Nunca en el proceso principal.
	•	Confirmación para acciones destructivas — el flag requires_confirmation=True en una herramienta permite interceptar antes de ejecutar (enviar emails, escribir en DB, hacer deploys).
	•	Auditoría centralizada — el registry loguea cada llamada con agent_id, tool_name y los parámetros. Esto permite detectar abusos y reconstruir el historial completo.
	•	Credenciales por agente — en producción, cada agente debería tener su propio API key / token con permisos restringidos en el servicio destino (ej. un rol de base de datos de solo lectura para el agente de datos).

Stack
	•	grammy → Telegram bot

	•	deepgram → Text To Speech
	•	OpenAI compatible API → LLM (usar diferentes modelos como Qwen3-32b, DeepSeek-V3, etc)

	•	tsx → ejecución en desarrollo
