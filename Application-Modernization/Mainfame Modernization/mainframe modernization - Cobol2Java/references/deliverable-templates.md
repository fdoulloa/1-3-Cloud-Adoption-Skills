# Deliverable Templates

## Existing Architecture Analysis

Include:

- repository and artifact inventory
- entry point list
- system context diagram
- DFD
- control flow topology
- data stores and external interfaces
- business rule catalog
- assembler/black-box assessment
- self-review and approval gate

## Cloud Native Design

Include:

- bounded context map
- service responsibility table
- service-owned data
- synchronous REST APIs
- asynchronous events or queues
- frontend component hierarchy
- database migration mapping
- security model
- observability model
- resilience and transaction consistency model
- self-review and approval gate

## API Contract Sketch

For each service, define:

- base path
- resource paths
- methods
- request fields
- response fields
- validation errors
- conflict/not-found semantics
- legacy-equivalence notes

## Implementation Report

Include:

- modules implemented
- files changed or generated
- business rules implemented
- frontend views implemented
- unit test coverage list
- command outputs summarized
- logical equivalence review
- gaps and stop conditions

## Completion Report

Include:

- Dockerfiles generated
- Kubernetes manifests generated
- CI/CD commands
- test coverage list
- local validation performed
- validation not performed and why
- production readiness gaps
- final acceptance status
